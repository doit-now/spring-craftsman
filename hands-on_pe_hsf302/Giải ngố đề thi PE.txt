I. ĐỀ THI CÓ GÌ?
* ĐỘ DÀI >= 5 TRANG 

* CUNG CẤP KÈM THEO 1 DATABASE SCRIPT (CHỨA LỆNH CREATE TABLE).TXT HOẶC .SQL
  (.TXT DOUBLE CLICK DEFAULT MỞ NOTEPAD
   .SQL DOUBLE CLICK DEFAULT MỞ SQL SERVER MANAGEMENT STUDIO NẾU CÓ CÀI)
  GIÚP CHÚNG TA CÓ CÁC LỆNH TẠO TABLE; NHƯNG KO CÓ LỆNH TẠO DATABASE
  
  - ĐỀ THI PE THEO STYLE DATABASE FIRST, PHẢI CÓ TABLE TRƯỚC
    (TRÊN LỚP MÌNH ĐÃ HỌC CODE FIRST - TỪ @ENTITY, CLASS SẼ THÀNH TABLE)

* ĐƯỢC THAM KHẢO TÀI LIỆU Ở MÁY LOCAL CỦA BẠN; 
* KHÔNG DÙNG LOMBOK (TUỲ TỪNG MÙA)

* CHÚ Ý CODING CONVENTION - QUY ƯỚC VIẾT CODE
  - ĐẢM BẢO ĐÚNG CHỮ HOA CHỮ THƯỜNG, SHIFT GẠCH _ NẾU CÓ KHI ĐẶT TÊN
  - ĐẶT ĐÚNG TÊN PROJECT (CÓ CHỮ HOA VÀ SHIFT GẠCH), ĐÚNG PROJECT CODE (MÃ DỰ ÁN) 
  - ĐẶT ĐÚNG TÊN PACKAGE, MÃ SỐ SINH VIÊN NHỚ HẠ XUỐNG CHỮ THƯỜNG (CHUẨN CÔNG NGHIỆP)

* ĐỀ CHỈ CÓ LIST, CREATE, SEARCH, DELETE (DÙNG MODAL KO DÙNG POPUP DEFAULT CONFIRM())
* TABLE CHÍNH: TOUR CÓ KEY TỰ TĂNG, 
               CỘT TOUR_NAME UNIQUE - CẤM TRÙNG, 
               CỘT NGÀY THÁNG NĂM LƯU Ý FORMAT NHẬP VÀ HIỂN THỊ dd/MM/yyyy
                          TRONG KHI ĐÓ DATABASE LƯU DẠNG yyyy-MM-dd
* CHÚ Ý PHẦN VALIDATION TRÊN CÁC Ô NHẬP - XEM BẢNG MÔ TẢ TRONG ĐỀ
* KHÔNG CÓ MỐI QUAN HỆ 1 - N NHƯNG VẪN CÓ 1 DROP-DOWN TREO ĐẦU DÊ LẤY THỊT HEO HARD-CODED
  STATUS (IN - INACTIVE, AC - ACTIVE, DR - DRAFT)

* NỘP BÀI: NÉN THƯ MỤC PROJECT THÀNH .ZIP (< 10MB) VÀ SUBMIT KHI THỜI GIAN LÀM CÒN LẠI 4-5 PHÚT

  SELECT * FROM TOURS (ĐỪNG QUÊN TÊN CLASS LÀ SỐ ÍT NHEN!!!)
