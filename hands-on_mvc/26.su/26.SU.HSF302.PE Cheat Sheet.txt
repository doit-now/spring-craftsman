II. KHAI BÁO CẤU HÌNH CSDL, PORT, VÀ THYMELEAF
* BẮT BUỘC PHẢI KHAI BÁO NGAY KẾT NỐI CƠ SỞ DỮ LIỆU TỚI SQL SERVER TRÊN MÁY MÌNH THÌ MỚI KHỞI ĐỘNG ĐC TOMCAT. 

* CÓ 2 LOẠI FILE LƯU THÔNG TIN CẤU HÌNH DỰ ÁN MÌNH
- application.properties (default - viết dàn ngang)

  server.port=6969  

- application.yaml (viết thụt ra vô)

  server:
    port:6969

1. KHAI BÁO PORT, CỔNG LẮNG NGHE REQUEST CỦA CON TOMCAT
- ĐỀ THI SẼ GHI RÕ PORT BAO NHIÊU, NẾU GHI 8080 (MẶC ĐỊNH) THÌ TA KO CẦN LÀM GÌ THÊM, KO KHAI BÁO LUÔN, 
  CÒN THÍCH THÌ GÕ RA:
  server.port=8080      # ghi chú: PHẢI thay bằng port khác nếu đề
                        #          KO DÙNG 8080

2. KHAI BÁO KẾT NỐI TỚI SQL SERVER 
- PHẢI LÀM NGAY VÀ LUÔN NẾU MUỐN CHẠY ĐC TOMCAT. KO CHẠY ĐC TOMCAT VÀ DB -> 0 ĐIỂM
- KHAI BÁO KẾT NỐI CSDL ĐỀ THI CHO GỢI Ý KHAI BÁO, TA CHỈ COPY & PASTE VÀO FILE CẤU HÌNH, CHỈNH LẠI PASSWORD CHO ĐÚNG LÀ OKIE
- KHI NỘP BÀI, SỬA LẠI PASSWORD NHƯ ĐỀ THI LÀ OKIE
>>>>> PHẢI TẠO TRƯỚC SẴN DATABASE MÀ TÊN DATABASE 100% GIỐNG ĐỀ THI; ĐỀ THI CHO SẴN TÊN DB
      HIBERNATE, JPA KO TỰ TẠO DATABASE, MÀ CHỈ TỰ TẠO TABLE TRONG DATABASE
>>>>> KO KẾT NỐI ĐC DB, THÌ KO KHỞI ĐỘNG APP THÀNH CÔNG, O ĐIỂM!!!

3. CHÚ Ý OPTION CODE FIRST
spring.jpa.hibernate.ddl-auto=create
- KHI LÀM BÀI THI, NÊN ĐỂ create ĐỂ ĐẢM BẢO TABLE ĐƯỢC TẠO TỪ CODE MỚI NHẤT, TỪ ENTITY MỚI NHẤT (DÀNH CHO CODE FIRST: TỪ CODE RA TABLE)
- TRƯỚC KHI NỘP BÀI, SỬA LẠI create THÀNH update GIỐNG NHƯ ĐỀ THI, NẾU ĐỀ THI LÀ 
  create THÌ ĐỂ NGUYÊN - TÚM LẠI GIỐNG ĐỀ 
- KHI ĐI LÀM THỰC TẾ, TRƯỚC KHI DEPLOY APP LÊN MÁY KHÁCH HÀNG, NHỚ SET OPTION update
  HOẶC LÀ TOANG DB KHÁCH
